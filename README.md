# Overview

This was developed as part of the **Human-Computer Interaction (IPM)** course at IST-UL.
The objective was to develop optimal user-centered solutions through continuous iteration (ideation, prototyping, evaluation) and validate it through competitive user testing (bake-offs).

# BakeOff1 (Quizzes-Tutor App Redesign)

This project focuses on creating a high-fidelity mobile UI prototype for redesigning the Quizzes-Tutor system,
improving the user experience of quiz creation, making it faster, more intuitive and more engaging.

## Deliverables, Techniques and Technologies

### First deliverable 

- Low-Fidelity Prototype (paper prototyping)
- Used techniques like Think-Aloud and Wizard-of-Oz to collect user feedback and continuously improve the UX/UI.

### Second deliverable

- High-Fidelity Prototype in [Figma](https://www.figma.com/education/higher-education/)

# BakeOff2 (Dense UI Selection)

The goal is to minimize the time required to select a target in a dense interface of 80 items (8×10 grid) by redesigning and optimizing the user interface.

The base system is implemented in p5.js, where users must quickly select highlighted targets from a grid.

- Use the [p5.js web editor](https://editor.p5js.org/)

## Technologies

- [p5.js](https://p5js.org/)
